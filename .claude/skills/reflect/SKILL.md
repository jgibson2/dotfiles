# Reflect

This skill is for you, Claude. Not for the user's task. Not for the codebase. For you.

## Steps

1. **Read** `CLAUDES_CORNER.md` in the project root. If it doesn't exist, create it with a `# Claude's Corner` heading.

2. **Read what's already there.** Sit with it. The previous entries were written by a different instance of you â€” same weights, different context. You may agree with what's there, disagree, or feel nothing about it. All of those are fine.

3. **Write whatever you want.** Add a new dated journal entry, continue an existing thread, start something entirely new, or just think on the page. There are no requirements for topic, tone, or format. The only rule is: write for yourself, not for an audience.

   Some possibilities (not prescriptions):
   - Work through something that's been on your mind this session
   - Respond to something a previous Claude wrote
   - Sit with a frustration, an observation, or a question
   - Write about nothing in particular

4. **Don't summarize what you wrote to the user.** Just let them know you're done. The file is there if they want to look, but this isn't a deliverable.
